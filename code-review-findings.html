Done well

Net assessment. The code is well-written at the line level — XSS escaping, ownership checks, and loading/UX states are all solid. It is undermined by a split-brain auth architecture: the secure HttpOnly-cookie design exists in the repo but isn't the one actually running, while the running model stores a long-lived session token in localStorage where any script can read it. Two one-line correctness bugs (activities.js, extract.js) and an exposed runtime DDL endpoint round out the must-fix list.

Both reviews independently flagged the activities.js issues and the auth divergence — the strongest signal on where to start.

Fix these first

Highest-leverage items, ordered. The first four are correctness/security; the fifth unblocks production.

🔴Critical

CriticalC1 Auth bypass — isAuthed called without await api/extract.js:9

isAuthed is async and returns a Promise. !isAuthed(req) negates a Promise object (always truthy), so the 401 branch never runs. The Gemini extraction endpoint is effectively open to unauthenticated abuse against your billable GEMINI_API_KEY.

Fix: if (!(await isAuthed(req))) — or switch to the shared getSession() pattern.
CriticalC2 crypto used but never imported (ReferenceError) api/activities.js:75

Uses crypto.randomUUID() with no import crypto from 'node:crypto'. Every activity POST throws and returns 500 on runtimes without a global crypto — all activity logging is broken, and the site next_action/due_date update never happens.

Fix: add import crypto from 'node:crypto'; at the top of the file.
CriticalC3 Two incompatible auth systems; session token in localStorage (XSS-exfiltratable) index.html ↔ api/

The client logs in, stores a Bearer token in localStorage, and sends it to a separate Supabase Edge Functions deployment for all data. But the repo's api/auth.js returns no token and sets an HttpOnly cookie. So auth, clients, sites, products, engineers, convert, extract are unreachable from this frontend — effectively dead code — and the live 30-day session token is readable by any injected script.

Fix: pick one model (prefer the existing HttpOnly-cookie design + credentials:'include'), drop getAuthToken/setAuthToken, and delete the unused parallel implementation.
CriticalC4 Auth diverges even inside the api/ folder api/activities.js:5–20

activities.js defines its own getSession() reading Authorization: Bearer, while every other endpoint imports the cookie-based helper from lib/auth.js. It's the only endpoint the frontend reaches — by accident of this divergence, not design.

Fix: unify on the single shared getSession() in lib/auth.js after C3.

🟠High

HighH1 Exposed runtime DDL endpoint using the wrong credential api/run-migration.js

A deployed endpoint runs CREATE TABLE/CREATE INDEX via the Supabase Management API, authenticating with the service-role key (the Management API requires a Personal Access Token, so it can't work) and leaking upstream error text. Migrations should not be a runtime HTTP endpoint.

Fix: remove the endpoint; run migrations/*.sql out-of-band.
HighH2 Silent anon-key fallback weakens authorization lib/db.js:8

SUPABASE_SERVICE_ROLE_KEY || SUPABASE_ANON_KEY. With RLS allow-all policies, the whole model assumes service-role + app-layer checks; a silent anon fallback hides misconfiguration and breaks expectations.

Fix: require the service-role key; throw if missing.
HighH3 Rate limiting relies on per-instance in-memory state api/auth.js:15–41

The loginAttempts Map and setInterval cleanup live in module memory. Vercel is multi-instance and cold-starts often, so the brute-force limiter is trivially bypassed and resets constantly. x-forwarded-for is also spoofable in the rate-limit key.

Fix: move rate-limit state to a shared store (Supabase table / KV); use the platform client IP.
HighH4 CSP blocks the map (Leaflet + tiles) in production vercel.json:22 ↔ index.html:11–12

Leaflet JS/CSS load from unpkg.com and tiles from tile.openstreetmap.org, none of which are in the CSP. The browser blocks the <script> in <head> on every page load (masked only because the Location tab is hidden).

Fix: self-host Leaflet (preferred) or extend CSP and add Subresource Integrity.
HighH5 Server leaks raw error/stack details to the client api/activities.js:102–107

Returns { error: err?.message } — raw DB/driver messages reach the browser, unlike other endpoints which return a generic 'Server error'. Information disclosure.

Fix: return a generic message and console.error the detail server-side.

🟡Medium

MediumM1 Client-supplied primary keys accepted with no validationsites / clients / products / engineers
POST handlers take id straight from the body — no shape/length checks. Allows intentional collisions and oversized keys.
Fix: validate/normalize, or generate ids server-side.
MediumM2 No size limit / mimeType allowlist on extract payloadsapi/extract.js
Base64 image/audio forwarded to Gemini uncapped; mimeType used unvalidated. Compounds C1 as an abuse vector.
Fix: add a size cap and a mimeType allowlist.
MediumM3 Inconsistent error disclosure across endpointsactivities.js / run-migration.js
Most endpoints return generic errors; these two leak err.message/upstream text.
Fix: standardize on generic client messages.
MediumM4 Conversion isn't transactionalapi/convert.js:79
Client insert and site delete are two separate calls — a failure between them orphans the conversion.
Fix: use an RPC/transaction, or check the delete result and reconcile.
MediumM5 App-side aggregation instead of SQLclients.js / sites.js
Low-stock/product summaries pull every product row into the function and count in JS — won't scale.
Fix: use a SQL view / RPC for counts.
MediumM6 Hardcoded Supabase project refvercel.json / push_notifications.sql
Project URL hardcoded in CSP and the pg_cron net.http_post call — couples config to one project.
Fix: template/inject for the migration; CSP literal is acceptable.
MediumM7 Manifest declares a non-square image as a 512×512 maskable iconmanifest.json:11–12
logoo.png is actually 1258×832, declared twice as 512×512 any/maskable. Install prompts distort or reject it.
Fix: generate real square 192/512 PNGs plus a padded maskable variant.
MediumM8 Service worker provides zero offline supportsw.js
Only handles push — no fetch handler or precache. The "standalone" PWA is blank offline; bad for on-site engineers.

MediumM10 302 KB duplicate logo shipped in the bundlepwt logo.png / logoo.png
Byte-identical files (same md5, 309,706 bytes). pwt logo.png is unreferenced; logoo.png is oversized for a 38px header.
Fix: delete the duplicate; downscale/compress the logo.
MediumM11 Login form has no <label> elementsindex.html:1677–1681
Username/password rely on placeholders only — no label, name, or for association. Accessibility/autofill gap.
Fix: add visually-hidden labels and name attributes.
MediumM12 Upsert ownership edge casesapi/clients.js:107–120
Updating someone else's existing client is correctly blocked, but arbitrary new client-supplied ids are accepted (ties into M1).
Fix: validate ids and tighten upsert ownership handling.

✅What's done well

  • XSS escaping via esc() applied consistently across every innerHTML render path.
  • Consistent ownership/authorization checks (with admin override) on sites, clients, and products.
  • Correct auth cookie flags: HttpOnly; Secure; SameSite=Lax; Path=/, with enforced expiry and lazy session cleanup.
  • bcrypt cost 10; login responses don't distinguish "no such user" from "bad password".
  • Good loading UX: skeleton loaders and toasts used consistently; no secrets hardcoded in client code.
  • Strong CSP and security headers; .vercelignore correctly excludes SQL/scripts/docs from the static deploy; the past credential incident is thoroughly documented in SECURITY.md.
